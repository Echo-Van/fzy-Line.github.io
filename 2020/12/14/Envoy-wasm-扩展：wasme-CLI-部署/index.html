Envoy Wasm 扩展：Wasm CLI 部署

发表于 2020-12-14 | 分类于 Cloud/Edge Computing |
学习了解 wasm - 面向未来的 Envoy 扩展方式。

　　WebAssembly 原本是一种可以使用非 JavaScript 编程语言编写代码并且能在浏览器上运行的技术方案。现在，它还可以被用于动态扩展 Envoy 代理，这为提升 Envoy 的可扩展性提供了更多的可能。但是，原生态集成 wasm 是十分繁琐的，需要手动处理依赖的工具链，ABI 版本验证保证兼容、访问权限控制等等。此外，在 Istio 中部署时也需要额外的操作，例如，挂在插件、创建 EnvoyFilter' 等。

　　因此，Solo.io 发布了 WebAssembly Hub ，它是一套用于构建、部署、共享和发现 Envoy 代理 的 Wasm 扩展工具和库。可以把它理解成 WASM 的 docker，一人构建四处分享，一处构建任意部署。实现了完全自动化开发和部署 Wasm 扩展所需的许多步骤。

wasme CLI

　Solo.io 还提供了一个强大的 CLI 以及一个易于使用的图形用户界面。 使用命令用户可以将构建的 Wasm 扩展上传到 Hub 注册中心，然后通过镜像的方式部署到 Istio。

　　构建完成后，会将镜像存储到：$HOME/.wasme/store/tag签名/ 目录下，其中包括以下四个文件：

  • descriptor.json ：定义文件的类型，wasm 文件的 sha256 签名，文件大小和文件名。

  • filter.wasm ：构建出来的 wasm 插件。

  • image_ref ：tag 标签字符串。

  • runtime-config.json ：从项目路径下拷贝过来的。

　　build 子命令的原理：调用 sdk 的编译工具，同时生成 descriptor.json 、image_ref 和 runtime-config.json 文件。通过查看wasme 的源码发现，它构建时使用的是 docker 的方式进行构建（即不需要自己去搭建构建环境）。
